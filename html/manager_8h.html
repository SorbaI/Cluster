Функция для формирования массива для передачи задач по сети.

Parameters
[in]num_tasksКоличество задач для выполнения.
[in]task_sizesУказатель на массив, содержащий размеры каждой задачи (в байтах).
[in]tasksУказатель на массив, содержащий данные задач.
Returns
Указатель на сформированный массив для передачи по сети. Возвращает NULL в случае ошибки.

Функция формирует структуру данных, пригодную для передачи задач по сети. Включает в себя информацию о размерах задач и сами задачи. Пользователь должен освободить выделенную память после использования.

Функция для инициализации структуры INFO_MANAGER.

Parameters
[out]managerУказатель на структуру INFO_MANAGER, которую необходимо инициализировать.
[in]addrСтрока, содержащая адрес для Управляющего узла (например, "127.0.0.1").
[in]portСтрока, содержащая номер порта для Управляющего узла (например, "8080").
[in]secondsМаксимальное время общего ожидания для Управляющего узла (в секундах).
[in]num_nodesКоличество рабочих узлов, необходимых для начала вычислений.

Функция инициализирует структуру INFO_MANAGER, устанавливая адрес прослушивания, максимальное время ожидания и требуемое количество рабочих узлов. После успешной инициализации поле is_init устанавливается в true.

int start_manager ( INFO_MANAGER *  manager,
size_t  num_tasks,
char *  tasks,
char *  ans

Функция для старта работы Управляющего узла.

Parameters
[in]managerСтруктура INFO_MANAGER, инициализированная функцией info_manager_init.
[in]num_tasksКоличество задач для распределенного вычисления.
[in]tasksУказатель на задачи для передачи по сети (результат работы create_task_structure).
[out]ansУказатель на область памяти, в которую последовательно записываются результаты выполнения задач.
Returns
Возвращает 0 в случае успеха, -EINVAL при некорректных аргументах и -1 при возникновении ошибок.

Функция ожидает подключения рабочих узлов в количестве, указанном в структуре INFO_MANAGER, получает информацию о количестве их ядер и распределяет задачи по принципу "одна задача - одно ядро".
